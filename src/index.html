Guide how to add/update source variables

Step 1
Open the "Data entry" menu-item by navigating to "Import data":
Step 2
Open the source variable table of your cohort:
In this example the DNBC cohort is selected.
Step 3
Click on the green + sign to add a row and click on the edit button in front of the row.
IMPORTANT: Make sure that when using both editing options (excel upload/molgenis edit), you should alter both, otherwise when updating your changes will become undone.
In this example edit row is clicked. When clicking "add row", the form will be empty.
Step 3
Now you can fill out the form and click on save.
IMPORTANT: Do not click in the grey area outside the modal, this causes your pop up to close.
Your variable is now added/updated. Nightly a mapping will be applied which will put your updated variable in the main source variable table, this means your variable will not instantly be updated in the main table.

Guide how to upload source variables

Step 1
Create an excel file with as tabname: LifeCycle_*here your cohort name*_SourceVariables.
Please note the casing, L, C, S and V are capitals.
Use the following format for your table:

variable label datatype values unit collectionType description dateOfUpdate dependencies
The name of the variable, cann't be null The label of the variable, cann't be null The type of the value, choose from Choose from these, , cann't be null The possible values for the variable Which unit is the variable in? Choose from these How was the information collected, cann't be null Describe of the variable, cann't be null Date that the variable was last updated for harmonization Description of dependencies of the variable on other variables

Step 2
Fill the file with your data.
Step 3
Go to the importer (Import data --> Upload data).
Step 4
Click on "Select a file..." and select your file. Then click the next button on the bottom right.

Possible errors

Solution:

Probably you made a mistake in naming the tab in excel. Note this is case sensitive and should be in the following format: LifeCycle_*cohort name*_SourceVariables. Click here for a list of all possible cohorts. This name is case sensitive as well.

Step 4
Select: Add entities / update existing. Click the next button on the bottom right again. The package step appears, since you are only uploading data you can ignore this and press next again. If you provided all columnheaders in your file, all fields will be green and you can press next again.

Possible errors

Solution:

You provided a date that is not formatted in a molgenis supported date format. Please make sure you use this format: yyyy-mm-dd.

Solution:

Description is a required column. Please provide a description for all rows.

Solution:

The cohort name you provided is not a valid cohort name. Please select one from this list. Again, the data is case sensitive, so make sure all casing is correct. Look at the "ID" column for this value.

Solution:

The datatype name you provided is not a valid datatype name. Please select one from this list. Again, the data is case sensitive, so make sure all casing is correct. Look at the "ID" column for this value.

For other errors, please contact us.

Your variable is now added/updated. Nightly a mapping will be applied which will put your updated variable in the main source variable table, this means your variable will not instantly be updated in the main table.

Step 6
Go to the importer (Import data --> Upload data).
Step 7

Possible errors

Solution:

You provided a date that is not formatted in a molgenis supported date format. Please make sure you use this format: yyyy-mm-dd.

Solution:

Target is a required column. Please provide a description for all rows. This error can also occur for other required columns.

Solution:

The cohort name you provided is not a valid cohort name. Please select one from this list. Again, the data is case sensitive, so make sure all casing is correct. Look at the "ID" column for this value. This error can also occur for the source and target variables.

For other errors, please contact us.
